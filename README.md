# 🤖 AIAnalyzer

> **ИИ-агент для автоматизированного анализа результатов тестирования студентов и выработки методических рекомендаций.** 

Агент обрабатывает выгрузки из систем дистанционного обучения (в форматах `.xlsx` и `.csv`), формирует матрицу результатов успеваемости и использует алгоритмы больших языковых моделей (LLM) для глубокого аудита: 4]. Нейросеть выявляет паттерны ошибок, анализирует качество формулировок тестовых вопросов и предлагает стратегические шаги по улучшению образовательного курса: 5].

---

## ✨ Ключевые преимущества

* **Гибкость инфраструктуры (Agnostic AI):** Поддержка как внешних высокоскоростных облачных API, так и полностью автономных локальных моделей (Local-first).
* **Безопасность:** Чувствительные образовательные данные могут обрабатываться внутри изолированного контура без доступа в интернет (при использовании локальной LLM).
* **Модульность:** Микросервисная архитектура позволяет масштабировать веб-интерфейс и вычислительный ИИ-кластер независимо друг от друга.
* **Умный парсинг данных:** Алгоритмы автоматического распознавания мульти-страничных Excel и CSV отчетов без строгой привязки к формату ячеек СДО.
* **UX/UI:** Интуитивно понятный интерфейс.

---

## 🛠 Технологический стек

Система построена на базе микросервисной архитектуры:

* **Язык и Фреймворк:** C# 12.0, ASP.NET Core 8.0/10.0 (MVC).
* **Frontend / Backend:** Обеспечивает веб-интерфейс, парсинг Excel-файлов, маршрутизацию и управление конфигурациями: 21]. Фронтенд реализован на HTML5, CSS3, Bootstrap 5 и jQuery (Unobtrusive Validation).
* **Работа с данными:** Используются библиотеки `ExcelDataReader` (для бинарных `.xlsx`) и `System.Text.Json`.
* **AI Engine (LLM Layer):** Изолированный сервер Ollama, предоставляющий REST API, совместимое с OpenAI.
* **Оркестрация:** Docker, Docker Compose.
* **Хранилище конфигураций:** Динамический JSON-файл `llm_providers.json`.

## 🔒 Безопасность и оптимизация

Агент спроектирован с учетом строгих требований к защите информации:

* **Интеграция государственных сертификатов:** В ядро системы (на уровне `Dockerfile`) вшит сертификат Национального удостоверяющего центра Минцифры РФ (`russian_trusted_root_ca.crt`). Это гарантирует надежное TLS/SSL-шифрование (защиту от MITM-атак) при обмене данными с отечественными провайдерами (GigaChat) и соответствие требованиям ФСТЭК.
* **Безопасное хранение ключей:** API-ключи хранятся в изолированном конфигурационном файле (`llm_providers.json`), защищенном правами доступа Unix (`chown app:app`) внутри контейнера.
* **Экономия токенов и защита от переполнения:** Встроенный алгоритм балансировки нагрузки автоматически разбивает массивные датасеты (более 40 вопросов) на блоки. Он генерирует промежуточные выводы и компилирует финальный отчет, что предотвращает переполнение контекстного окна (Context Overflow) и экономит платные токены.

## ⚙️ Системные требования

Требования зависят от выбранного режима работы нейросети:.

### Вариант А: Внешние API (Groq, GigaChat, OpenAI)
Основные вычисления происходят на стороне провайдера.
* **CPU:** Любой современный 2-ядерный процессор (от 2.0 ГГц).
* **RAM:** От 4 ГБ.
* **Сеть:** Стабильное широкополосное подключение к интернету.

### Вариант Б: Локальная нейросеть (Ollama + Llama 3 8B)
Высший уровень безопасности: данные не покидают контур организации, но требуются мощности для инференса.
* **CPU:** Уровня AMD Ryzen 5 3600 / Intel Core i5 10-го поколения или выше.
* **GPU:** От NVIDIA GTX 1660 Ti (строгий минимум 6 ГБ VRAM) для аппаратного ускорения (CUDA).
* **RAM:** 16 ГБ и более.
* **Диск:** SSD-накопитель, минимум 10 ГБ свободного места (3 ГБ под базовый образ оркестратора + 4.7 ГБ под веса модели Llama 3).

## 🚀 Развертывание (Инструкция для администратора)

Управление подключенными нейросетями физически хранится в файле `AIAnalyzer/llm_providers.json`. Доступно два способа развертывания.

### Способ 1. Docker Compose (Рекомендуемый)
Гарантирует воспроизводимость окружения и изолирует ИИ от хост-системы[cite: 45, 46]. Убедитесь, что установлен Docker Desktop и порты `8080` и `11434` свободны.

Выполните следующие команды для клонирования репозитория, сборки образов и инициализации локальной нейросети в изолированном томе:

```bash
# 1. Клонирование репозитория
git clone [https://github.com/LittleSangwoo/AIAnalyzer.git](https://github.com/LittleSangwoo/AIAnalyzer.git)

# 2. Сборка и запуск контейнеров (выполнять в директории с docker-compose.yml)
docker-compose up -d --build

# 3. Загрузка весов модели Llama 3
docker exec -it aianalyzer-ollama-server-1 ollama run llama3
```
> 💡 **Примечание:** для проверки нахождения в нужной директории выполните команду `ls` и найдите файл `docker-compose.yml`.

### Способ 2. Локально (Visual Studio + Native Ollama)
Используется для разработки и отладки проекта.

1. Установите **Ollama** с официального сайта и скачайте нужную модель через терминал:
   ```bash
   ollama run llama3
   ```
2. Оставьте сервер Ollama работать в фоновом режиме (иконка в трее).
3. Откройте файл `AIAnalyzer.sln` в **Visual Studio**, выберите профиль запуска **AIAnalyzer** (`https`/`http`, не Docker) и запустите проект.

## 📖 Руководство пользователя

### Работа с документами
1. На главной странице загрузите файл формата `.xlsx` или `.csv` с результатами тестирования.
2. Система отобразит **Матрицу результатов**, подсветив проблемные вопросы красным цветом *(критическая зона, много ошибок)*.
3. Нажмите кнопку **«Аудит курса»** для запуска ИИ-анализа.

> ⚠️ **ПРЕДУПРЕЖДЕНИЕ О ЛИМИТАХ:** При работе с бесплатными внешними API (например, Groq) существуют строгие лимиты на количество токенов в минуту (TPM). Если вы загружаете массивный файл на несколько тысяч ответов, облачное API может отклонить запрос из-за превышения объема текста. В таких случаях настоятельно рекомендуется переключаться на локальную нейросеть, не имеющую ограничений по TPM.

### Настройка Endpoints и Моделей
При добавлении новых нейросетей на странице «Настройки» обращайте особое внимание на ключевые поля:

* 🌐 **Endpoint (Шлюз API):** Точный URL-адрес сервера, принимающего запросы.
  * *Внешние (Groq):* `https://api.groq.com/openai/v1/chat/completions`
  * *Докер-сеть:* `http://ollama-server:11434/v1/chat/completions`
  * *Windows (Нативная Ollama):* `http://127.0.0.1:11434/v1/chat/completions` (или `http://host.docker.internal:11434/v1/chat/completions`, если сайт развернут в Docker).

* ⚙️ **ModelName:** Технический идентификатор нейросети на сервере. Это **самое критичное поле** — любая опечатка вызовет ошибку!
  * *Локальные:* Узнать имя можно через команду `ollama list` в консоли (например, `llama3`).
  * *Внешние:* Копируется из личного кабинета разработчика API (например, `llama-3.3-70b-versatile` для Groq).

## 🧠 Поддерживаемые модели ИИ (AI & LLM Layer)

**AIAnalyzer** использует мульти-агентный подход с динамическими системными промптами (System Prompts). В зависимости от задачи агент применяет разные роли: *Методист, Тестолог, Аналитик*.

### Рекомендуемые локальные сети (Высокая приватность)
* **Saiga Llama 3** — *Настоятельно рекомендуется!* Дообучена на качественных русскоязычных текстах, идеально понимает русскую грамматику и методические термины.
* **Qwen 2.5 (7B / 14B)** — Открытая модель от Alibaba с феноменальными результатами в логическом анализе данных. Быстро работает на средних GPU.
* **Mistral-Nemo (12B)** — Отличный компромисс между скоростью и объемом контекста для загрузки массивных таблиц.

### Рекомендуемые внешние API (Скорость и мощность)
* **Groq API (Llama-3-70B)** — Сверхбыстрая обработка (LPU acceleration) до 800 слов в секунду. *Примечание: требует VPN на территории РФ.*
* **GigaChat (Сбер)** — Глубокий семантический анализ на русском языке и полное соответствие 152-ФЗ. Оптимальный выбор для госвузов.
* **YandexGPT** — Резервный канал для работы с огромными контекстами (RAG-интеграции) и удобной тарификацией.

## 🔧 Устранение типичных неполадок

| Проблема | Возможная причина | Решение |
| :--- | :--- | :--- |
| **Подключение отвергнуто**<br>`(127.0.0.1:11434)` | Имя модели в настройках не совпадает с тем, что скачано на сервер. | Проверьте поле «Имя целевой модели». Запустите `ollama list` для сверки. |
| **Ошибка: Connection refused** | Веб-сайт не видит сервер Ollama. Порт закрыт или неверный IP. | Проверьте `Endpoint` в настройках. Убедитесь, что Ollama запущена. |
| **Не сохраняются новые настройки** | Проблема с правами доступа к `llm_providers.json` внутри Docker. | Убедитесь, что в `Dockerfile` корректно прописана команда `chown` для выдачи прав приложению. |
