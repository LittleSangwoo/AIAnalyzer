🧠 Задача 1: Фабрика Промптов и Формирование Контекста
Тебе нужно написать логику, которая будет принимать готовую статистику (которую посчитала напарница), отрезать всё ненужное и превращать в текст, понятный нейросети.

Что делать: Написать класс AiPromptBuilder, который берет список QuestionStatDto, фильтрует только ErrorZone.Red, и склеивает их в текстовый блок: "Вопрос: [текст], Ошибок: [кол-во]".

Переключатель ролей: Реализовать логику (switch), которая в зависимости от кнопки с фронтенда (Compare, Critical, Report) подставляет нужную системную инструкцию в начало запроса.

🤖 Задача 2: Интеграция с API (Собственно, Агенты)
Создать сервисы, которые будут физически стучаться в нейросети.

Что делать: Написать реализации интерфейса IAiAgentService для вызова GigaChat и DeepSeek через HttpClient.

Настройки: Вынести API-ключи в appsettings.json, чтобы их можно было легко менять. Настроить JSON-десериализацию ответов от нейросетей.

📊 Задача 3: Генератор Excel-отчетов (ClosedXML)
Заказчик хочет получить готовый артефакт в виде файла. Это твоя зона ответственности.

Что делать: Написать ReportGeneratorService. Этот класс будет брать полную таблицу данных и текстовый ответ от ИИ, и собирать .xlsx файл.

Визуал: С помощью библиотеки ClosedXML программно красить строки в таблице (красный/желтый/зеленый) и выводить текст нейросети на отдельный лист. Результат должен возвращаться в виде массива байтов (byte[]), чтобы браузер мог его скачать.

🔌 Задача 4: Контроллеры API
Тебе нужно сделать "ручки" (endpoints), за которые будет дергать фронтенд напарницы.

Что делать: В твоем AnalyzerController добавить два новых метода:

[HttpPost("analyze-ai")] — принимает данные с фронта, отправляет в нейросеть, возвращает текст анализа.

[HttpPost("download-report")] — принимает данные + текст от ИИ, генерирует Excel и возвращает файл File(bytes, "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet", "Report.xlsx").
